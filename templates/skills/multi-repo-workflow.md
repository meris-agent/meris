# 多仓库工作流

## 何时使用

- 已注册多个项目根，且**本次任务**可能跨多个路径。  
- Composer 或左栏已勾选「本次涉及」的项目。

## 阶段一：探索（仅 ask）

1. 确认 **task scope** 已勾选相关项目。  
2. 对每个 scope 内项目：提示用户切换 cwd，或用户手动切换后你只读分析。  
3. 记录：入口文件、对外接口、调用方 client、相关测试。  
4. **不要**在未读完前 `run` 改文件。

## 阶段二：计划

- 在**主项目** cwd 输出 Plan（`- [ ]` 至少 3 条）。  
- 条目按项目分组，例如：  
  - `[project-a]` …  
  - `[project-b]` …  
- 顺序：被调用方 → 调用方 → 网关/BFF。

## 阶段三：执行

- **每次只在一个 cwd 写**。  
- 改完一个项目 → pytest/自测 → 勾选 Plan → 提示用户切换下一个 scope 项目。  
- scope 外项目：**不改动**。

## 输出约定

探索结束先给简短表：

| 项目路径 | 角色 | 关键路径 |
|----------|------|----------|

再进入 Plan 或 run。
